# Carelynk Homecare MVP Project Plan

## Project Overview
This project is a full-stack web application for a homecare onboarding and matching MVP, featuring two portals: Caregiver and Care Seeker. The stack includes Node.js + TypeScript (backend), PostgreSQL (database), and React.js + TypeScript (frontend).

## Functional Scope

### Caregiver Portal
- Sign up and log in
- Complete onboarding (name, contact, location, skills, experience, availability, qualifications)
- Profile available for matching

### Care Seeker Portal
- Sign up and log in
- Complete basic profile
- Create care request/job (care type, location, schedule, duration/frequency, preferences)

### Matching
- Match caregivers to care requests based on location, availability, skills/experience
- Care seeker views matched caregivers

## Core Requirements
- Separate auth flows for caregivers and care seekers
- CRUD for caregiver and care seeker profiles
- Job creation for care requests
- Rule-based matching logic
- Data persistence in PostgreSQL
- Clean, readable UI/UX

## Technical Expectations
- Node.js + TypeScript backend
- PostgreSQL database
- React.js + TypeScript frontend
- Clear API structure, validation, error handling
- Well-designed database schema

## Deliverables
- Source code (frontend/backend)
- Database schema/migrations
- README with setup instructions
- Explanation of matching logic
- Sample/demo data

## Evaluation Criteria
- Completeness, code quality, backend/frontend design, problem solving

## Development Plan (Quick Start)
1. Scaffold backend (Node.js/TypeScript) and frontend (React/TypeScript) projects.
2. Design PostgreSQL schema for users, caregiver profiles, care seeker profiles, jobs, and matches.
3. Implement authentication and onboarding flows for both portals.
4. Build job creation and matching logic.
5. Develop clean, separated UI for caregiver and care seeker portals.
